# analysis-reporting

分析結果、実験成果、モデル評価などをまとめる際に参照するスキルです。

---

## 言語

指定がない限り、レポートは**日本語**で作成する。
コード・変数名・カラム名は英語のままとする。

---

## レポートの推奨構成

### 1. 結論（冒頭に配置）

読者が最初に目にするべき主要な発見を先に書く。

```markdown
## 結論

2024年Q1の売上は前年同期比 +12.3%（95%CI: +9.8%〜+14.8%）となった。
伸びの主因はエンタープライズセグメントの新規獲得であり、
SMBセグメントは前年比 -3.2% と微減した。
```

### 2. 事実

データから得た客観的な情報を記述する。

```markdown
## 事実（データが示すこと）

- 総売上: 1億2,340万円（前年同期: 1億990万円）
- 新規顧客数: 234社（前年同期: 198社、+18.2%）
- 解約率: 4.2%（前年同期: 4.8%）
- 分析対象期間: 2024-01-01 〜 2024-03-31
- 分析対象: 有効契約中の全顧客 (n=1,847)
```

### 3. 仮定

分析時に設定した前提条件を明記する。

```markdown
## 仮定

- 試験導入中（トライアル）の顧客は除外した
- キャンセル後30日以内の再契約は継続とみなした
- 為替レートは2024年3月末の固定レートを使用した
```

### 4. 解釈

データから導かれる示唆と意味合いを記述する。

```markdown
## 解釈

エンタープライズ案件の増加はQ4の営業強化施策の遅延効果と考えられる。
SMBの微減は市場全体の傾向と一致しており、自社固有の問題ではない可能性が高い。
```

### 5. 制約・注意点

結果の限界とバイアスの可能性を明記する。

```markdown
## 制約・注意点

- 2月の障害（3日間）の影響を除外できていない
- 地域別の分析は次フェーズで実施予定
- 因果関係ではなく相関関係の分析である
```

---

## 記載が必要なコンテキスト情報

| 項目 | 記載内容の例 |
|------|------------|
| データ期間 | 2024-01-01 〜 2024-03-31 |
| フィルター条件 | status = 'active', country = 'JP' |
| サンプルサイズ | n = 1,847 顧客 |
| 指標の定義 | 売上 = 請求確定ベース（キャンセル分を除く） |

## 再現性のための記録

```markdown
## 再現方法

**入力データ:**
- `data/raw/sales_2024q1.parquet`（2024-04-02 時点のスナップショット）

**実行スクリプト:**
- `notebooks/001_sales_analysis_2024q1.ipynb`
- `uv run papermill notebooks/001_sales_analysis_2024q1.ipynb outputs/reports/...`

**出力ファイル:**
- `outputs/figures/monthly_revenue_2024q1.png`
- `outputs/tables/segment_summary_2024q1.csv`
```

## トーンのバランス

- **ステークホルダー向け**: 専門用語を避け、ビジネスへの影響を先に書く
- **技術レビュアー向け**: 分析手法・前処理・モデルの詳細を別セクションに記載する
- **不確実性を正直に伝える**: 「示唆される」「可能性が高い」など確度を示す表現を使う
