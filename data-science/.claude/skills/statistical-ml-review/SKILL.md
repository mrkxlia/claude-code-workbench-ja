# statistical-ml-review

統計分析、仮説検定、A/Bテスト、機械学習モデルの構築・評価・特徴量エンジニアリング時に参照するスキルです。

---

## 分析開始前に明確化する項目

作業を始める前に以下をすべて確認・文書化する。

| 項目 | 内容 |
|------|------|
| 目的変数 | 何を予測・説明したいか |
| 分析単位 | 1行が何を表すか（ユーザー / セッション / 注文など） |
| 対象期間 | いつからいつまでのデータを使うか |
| サンプル定義 | 誰を・何を対象にするか |
| 除外基準 | 何を除外するか（理由とともに） |
| 前提条件 | 分析が成立するための仮定 |
| データ漏洩リスク | 予測時点以降の情報が特徴量に含まれないか |
| 評価メトリクス | 何で成功を測るか（ビジネス指標との対応） |
| ベースライン | 比較する基準（単純な予測・既存モデルなど） |

---

## 統計分析の原則

- **相関と因果を区別する** — 観察データから因果を主張する場合は明示的に注意書きをする
- **信頼区間を提示する** — 点推定だけでなく不確実性を示す
- **過度な主張を避ける** — データで支持される範囲に結論を留める
- **実験設計を文書化する** — 事前登録・検出力分析・多重検定の補正

## 機械学習の原則

```python
from sklearn.model_selection import train_test_split

# データを分割する（時系列の場合は時間順に分割する）
df_train, df_test = train_test_split(
    df, test_size=0.2, random_state=42, stratify=df["target"]
)

# 前処理はtrain setのみでfitしてtest setにtransformする（データ漏洩の防止）
from sklearn.preprocessing import StandardScaler

scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)  # fit はtrainのみ
X_test_scaled  = scaler.transform(X_test)       # testにはtransformのみ
```

- **訓練・検証・テストを分離する** — テストデータは最終評価のみに使う
- **前処理パイプラインを文書化する** — 特徴量生成の手順を再現可能にする
- **単純なベースラインと比較する** — 複雑なモデルが本当に必要かを検証する
- **交差検証を検討する** — 小さなデータセットでは特に重要
- **モデルの限界を報告する** — OOD（分布外）データへの注意点を明記する
