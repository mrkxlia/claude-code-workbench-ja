# codex-implement — Codex に実装を依頼する（/codex-implement）

実装を **OpenAI Codex** に代行させるスキルです。ユーザーは Codex を直接操作しません。
Claude Code が Codex CLI を**非対話モード**（`--sandbox workspace-write`）で駆動し、
**Codex がワークスペースのファイルを直接編集**します。その後、Claude が差分とテストを
**検証**して要約します。実際の codex 実行は **`codex-implementer` サブエージェント**に
委譲し、冗長な出力をメインの文脈から隔離します。

## 使い方

- `/codex-implement <実装タスクの説明>` — タスクを Codex に実装させる
- `/codex-implement <タスク> @path/to/file` — 追加で渡したいファイルを明示指定

「Codex に実装させて」のような自然文でも発動します。

## 前提

`codex` CLI が導入・認証済みであること（`codex-bridge/README.md` の「前提」参照）。
未導入・未認証の場合、サブエージェントが日本語で案内して終了します。

## フロー

1. **タスク・制約・対象範囲を整理する**
   - 何を作るか、触ってよい範囲、踏襲すべき既存パターン、成功基準を明確にする。

2. **明らかに必要なファイルを用意する**（既定動作）
   - 編集対象ファイル＋近接の既存実装・呼び出し元・型定義/インターフェース＋
     該当する CLAUDE.md / AGENTS.md の抜粋を特定し、**内容ごと**渡す。
   - Codex にパスを名指しして開かせるのではなく、**Claude が内容を渡し切る**。
   - 大きすぎる場合は「全文 → 関連抜粋 → パス名指し」の順に降格（バイト数上限を意識）。

3. **`codex-implementer` を起動する**（Task ツール）
   - タスク・制約・**同梱するファイル内容**を渡す。
   - サブエージェントが定番型プロンプトを組み、Codex を workspace-write で実行
     （Codex が直接編集）→ 差分とテストを検証して要約を返す。

4. **結果を提示する**
   - 変更ファイル一覧・実装要約・テスト結果・未解決事項を見せる。
   - ネットワーク/依存が必要で失敗した場合は、その旨と対処（利用者が緩い
     サンドボックスを選ぶのは利用者責任）を添える。

5. **セルフレビューを提案する**
   - 続けて `/codex-review` で Codex に自分の変更をレビューさせることを提案する。

## non-git 環境について

git を使っていないプロジェクトでも動作します。検証は `git diff` の代わりに、Codex の
「変更ファイル一覧」報告を `Read` で突合する方式にフォールバックします。

## このスキルがやらないこと

- ユーザーに codex を手動で叩かせる（すべて Claude が非対話で駆動する）
- 危険なサンドボックスフラグを使う（workspace-write 既定、`danger-full-access` は不使用）
- テスト・型チェックを確認せず「完了」と報告する
