# ask-claude — 別の Claude に相談する／セカンドオピニオン（/ask-claude）

設計の是非・代替案・デバッグ方針・トレードオフなど、**自由形式の相談**を **別の Claude インスタンス**に
投げて独立した意見をもらうスキルです。ユーザーは CLI を直接操作しません。Claude Code が
`claude` CLI を**非対話モード**（`claude -p`）で、**編集不可（plan モード）＋読み取り系ツールのみ**に
制限して起動し、回答を要約して提示します。実際の実行は **`claude-advisor` サブエージェント**に
委譲し、冗長な出力をメインの文脈から隔離します。

## 使い方

- `/ask-claude この設計（〜）は妥当？代替案は？`
- `/ask-claude なぜこのテストが落ちるか方針を相談したい @path/to/test`

「別の Claude に聞いて」「もう一人の意見も」のような自然文でも発動します。

## 他スキルとの違い

- **peer** … 内部 Claude サブエージェントで完結（**外部 CLI 不要・git 不要**）。依存を増やしたくない・
  オフライン/git なし環境ならこちら。ask-claude は**別プロセスの Claude**を起動して独立性を高める点が違う。
- **/codex-ask（codex-bridge）** … 相談相手が **Codex**（`codex` CLI）。別エンジンの視点が欲しいとき。
- **/code-review・/codex-review** … 行レベルのコードレビュー。ask-claude は自由形式の助言で、コードは書かない。

## 前提

`claude` CLI が導入・認証済みであること（`command -v claude` で確認）。未導入・未認証の場合、
サブエージェントが raw なエラーを出さず日本語で案内して終了します。

## 安全方針（読み取り専用の強制）

`claude-advisor` は **`--permission-mode plan`（編集不可）を第一ガード**、`--allowedTools` で
読み取り系ツールに限定する多層防御で起動します（意図しないファイル編集を実フラグで防ぐ）。
危険な権限スキップフラグ（`--dangerously-skip-permissions` 等）は使いません。

## フロー

1. **相談内容を整理する** — 何を判断したいか、選択肢、制約を明確にする。
2. **明らかに関係するファイル内容を用意する** — 相談に必要な抜粋を渡し切る
   （大きすぎる場合は関連箇所に絞る）。
3. **`claude-advisor` を起動する**（Task ツール） — 相談内容と同梱ファイルを渡す。
   サブエージェントが別 Claude を読み取り専用で実行し、回答要約を返す。
4. **結果を提示する** — 結論・根拠・推奨アクション・留意点を見せる。
5. **必要なら橋渡しする** — 方針が固まったら実装や `/code-review`、別エンジンの意見が欲しいなら
   `/codex-ask` を提案する。

## このスキルがやらないこと

- ユーザーに CLI を手動で叩かせる（すべて Claude が非対話で駆動する）
- リポジトリのファイルを編集する（読み取り専用を実フラグで強制）
- 危険な権限スキップフラグを使う
- 別 Claude の生出力をそのままメインに垂れ流す（必ず要約する）
