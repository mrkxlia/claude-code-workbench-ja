# /export — Codex / Kiro へ生成する（Track A）

このスキルは `multi-model-dist/` のジェネレータを起動するための**ローカル作業用**スキルです（プラグイン配信物ではない）。

## 何をするか

`generators/bin/export.sh` を呼び、原本 `*/.claude/**`・`CLAUDE.md` から **MAPPING.md の Track A allowlist** に
従って Codex / Kiro の生成物を `multi-model-dist/build/` に書き出します。**原本は一切編集しません。**

```bash
multi-model-dist/generators/bin/export.sh --target codex,kiro   # 既定
multi-model-dist/generators/bin/export.sh --target codex        # Codex のみ
```

## 不変条件（必ず守る）

- 走査はセクション配下の `<section>/.claude/**` のみ。**ルート直下 `.claude/`・`.claude-plugin/` は除外**。
- 生成物にはセンチネル（`GENERATED by multi-model-dist — DO NOT EDIT. source: ...`）。
  **手書き（センチネル無し）の生成先は上書きしない。**
- 冪等：同じ原本から2回生成して差分ゼロ。`git status` で原本 `.claude/**` に差分が出ない。

## 検証

```bash
python3 multi-model-dist/generators/lib/test_convert.py   # ゴールデン/往復テスト（ALL GREEN を確認）
```

## 範囲（MAPPING.md 準拠）

- **生成する（Track A）**: T1 スキル（notes / spec-extract / create-plan* / data-science 10種 /
  model-setup の task-brief・backlog-loop・pr-merge・long-run）、T2p スキル＋エージェント対
  （peer＋peer-engineer / fan-out・verify-fresh＋model-setup エージェント3種 / review-panel＋panel-* 4種。
  サイドカーの personas.md・SPEC.md 等も複製）、ガイダンス CLAUDE.md（global-claude-md-sample・data-science＝
  Codex+Kiro、model-setup＝Kiro のみ）。
- **生成しない（Track B / 対象外）**: パイプライン・self-improve・knowledge-share・codex-bridge・フック・ask-claude。
  これらは `reimpl/SPEC/` を共有源に各ツールへ手書き再実装する。

## 依存

`python3`（標準ライブラリ）＋ `PyYAML`。未導入なら `pip install pyyaml`。
TOML は内蔵シリアライザで出力し `tomllib` で往復検証する（外部 TOML ライブラリ不要・決定的出力）。
