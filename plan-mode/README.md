# plan-mode — 変更せず「調査して計画を立てる」スキル集

AI エージェントに、**変更を加える前に「まず理解して計画する」モード**を持たせるための Claude Code スキル2種＋
要件定義です。Codex の Ask モードや Claude / Cline の Plan モードに相当する挙動を、特定ツールに依存しない
Agent Skill として実装しています。コーディングに限らず一般タスクにも使えます。

- **create-plan** — 変更を一切せず、ローカル / Web / MCP / ユーザ確認で情報を集め、別の AI エージェントが
  そのまま実行に移せる粒度の**実行計画**を1つの計画ファイルに書き出すスキル。
- **create-plan-calibrate** — `create-plan` を、実際に動かす AI エージェントのツール・シェル・MCP・仕様に
  合わせて**較正（調整）**するセットアップスキル（手動起動のみ）。
- **SPEC.md** — `create-plan` の要件定義（単一の真実源）。不変要件 `INV-n` と環境依存の調整ポイント `ADJ-n` を
  ID 付きで定義します。`create-plan` の SKILL.md はこの ID を参照し、`create-plan-calibrate` は ADJ-n を実値で埋めます。

---

## なぜ2本に分けるのか

「変更しない」の境界（どのツール・コマンドが副作用を持つか）は、実際に動かすエージェントが使える
**ツール名・シェル・MCP に依存**します。それらが不明なままでも安全に動くよう、`create-plan` は
環境依存の箇所を `<<ADJ-n: …>>` という目印にし、未確定時は **Read / Grep / Glob と明白な読み取り MCP のみ**を使う
「較正前フォールバック」で動きます。`create-plan-calibrate` がエージェントの仕様を調べてこの目印を実値に置き換えると、
そのエージェントに最適化された状態になります。

## ファイル構成

```
plan-mode/
├── README.md                                   # このファイル
├── SPEC.md                                     # create-plan の要件定義（INV-n / ADJ-n）
└── .claude/skills/
    ├── create-plan/SKILL.md                    # スキル1: 調査・計画（変更しない）
    └── create-plan-calibrate/SKILL.md          # スキル2: create-plan を対象エージェントに較正
```

---

## 使い方

### 1. ファイルをコピーする

コピーするのは **`.claude/skills/` 配下の2スキルと `SPEC.md`** です。`plan-mode/` というラッパディレクトリごと
持ち込まないよう注意してください（パスが二重になります）。

```bash
# 自分のプロジェクトのルートから実行する
cp -r path/to/plan-mode/.claude ./.claude         # 既に .claude があれば skills/ 配下だけマージ
cp path/to/plan-mode/SPEC.md ./.claude/skills/create-plan/SPEC.md
```

`SPEC.md` は `create-plan` スキルから参照できる場所（同梱ディレクトリなど）に置いてください。

### 2. （任意）対象エージェントに較正する

`create-plan` を Claude Code 以外の内作エージェントで使う場合や、使えるツール・MCP を絞り込みたい場合は、
較正スキルを手動起動します。

```
/create-plan-calibrate
```

エージェントのツール・シェル・MCP・Web 可否などを調査し、承認の上で `create-plan/SKILL.md` の
`<<ADJ-n: …>>` を実値に置き換えます。較正しなくても `create-plan` は安全側（較正前フォールバック）で動きます。

### 3. 計画を立てる

```
/create-plan <達成したいゴール>
```

または「変更しないで計画だけ立てて」「いきなり実装しないで方針をまとめて」のような依頼でも発動します。
情報収集 → 不明点の確認 → `<slug>-plan.md` への計画書き出し、の順に進み、**変更は一切せず**に停止します。
生成された計画ファイルは、別のエージェント/セッションがそのまま実行に移せます。

---

## 設計上の約束（SPEC.md より）

- `create-plan` は**計画ファイル以外を一切変更しない**（INV-1 / INV-2）。
- 情報は4系統（ローカル / Web / MCP / ユーザ質問）から集め、`[確定]` / `[推定]` / `[不明]` のラベルを付す（INV-3）。
- 計画は実行可能な粒度（ステップ ID・対象・アクション・検証・依存）で書く（INV-5）。
- スクリプトは計画ファイル内のコードブロックとしてのみ記述し、独立ファイル作成・実行はしない（INV-7）。
- 計画を書き出したら停止し、自分は実装に移らない（INV-6）。
