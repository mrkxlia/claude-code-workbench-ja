# SPEC.md — `create-plan` スキルの要件定義（単一の真実源）

このファイルは `create-plan`（調査・計画スキル）が満たすべき要件を定義する**契約**です。
`create-plan/SKILL.md` は不変要件を再記述せず、ここで定義する **INV-n** を ID で参照します。
`create-plan-calibrate` は、ここで定義する **ADJ-n**（調整ポイント）を該当 AI エージェントの実値で埋めますが、
**INV-n は決して変更・緩和してはなりません**。

---

## 不変要件 INV-n（環境に関わらず常に守る）

| ID | 要件 |
|----|------|
| **INV-1** | 計画ファイル**以外**への書き込みを禁止する。コード・設定・ドキュメントを変更しない。シェル経由のファイル出力（リダイレクト `>`・`Out-File`・`Set-Content` 等）も禁止。 |
| **INV-2** | 副作用のあるコマンド・ツール実行を禁止する（ファイル生成/移動/削除、git の変更系、パッケージインストール、外部への送信・投稿など）。読み取り専用の調査のみ行う。 |
| **INV-3** | 情報は4系統（ローカルファイル/コードベース、Web、MCP、ユーザへの質問）から集め、各情報に確信度ラベル `[確定]` / `[推定]` / `[不明]` を付す。 |
| **INV-4** | 推測で要件・前提を埋めない。重要な不明点はユーザに確認する（確認手段は ADJ-5）。 |
| **INV-5** | 最終成果物は、別の AI エージェント/セッションがそのまま実行（修正・移行）に移せる**実行可能な粒度**の計画ファイル1本とする。 |
| **INV-6** | 計画完成後はそこで**停止**し、自分は実装・実行に移らない。実行は別セッション/別承認に委ねる。 |
| **INV-7** | スクリプト（PowerShell / Python 等）は**計画ファイル内のコードブロックとしてのみ記述**してよい。`.ps1` / `.py` 等の独立ファイルとして作成すること、およびその実行は禁止する。 |

### 確信度ラベルの定義（INV-3）

- `[確定]` — 一次証拠（実ファイル・実データ・ユーザの明言・公式ドキュメント）で裏付けられる。
- `[推定]` — 根拠はあるが未確認。読み手が検証すべき。
- `[不明]` — 情報が得られていない。ギャップとして明示する。

---

## 調整ポイント ADJ-n（環境依存。`create-plan-calibrate` が実値で埋める）

`create-plan/SKILL.md` 内では各調整ポイントを `<<ADJ-n: 既定の説明>>` の目印で示す。
較正が未実施でも、`create-plan` は「較正前フォールバック」（SKILL.md の中核ルール参照）で安全側に動く。

| ID | 埋めるべき内容 |
|----|----------------|
| **ADJ-1** | 実行環境のシェル種別（PowerShell / コマンドプロンプト / その他）と、読み取り専用で使える確認コマンドの例。 |
| **ADJ-2** | 使用可能な**情報取得ツールの具体名**（ローカル読取・Web 取得・MCP の読み取り系ツール名）。 |
| **ADJ-3** | **禁止すべきツール・コマンドの具体名**（書き込み・破壊・送信・git 変更系・MCP の書き込み系）。 |
| **ADJ-4** | **Web 取得の可否**（オフラインの内作エージェントなら無効化し、Web 系統をスキップする）。 |
| **ADJ-5** | ユーザへの質問手段（AskUserQuestion 相当の有無）。無い場合は**計画ファイルに質問を残して人間へ返す**縮退動作を採る。 |
| **ADJ-6** | 計画ファイルの既定パス／命名規約。既定は `<slug>-plan.md`（`<slug>` はゴール文字列を小文字化し英数字とハイフンに正規化した短い識別子）。 |
| **ADJ-7** | そのエージェントがツール許可リスト（`allowed-tools` 等）に対応するか、対応する場合の設定方法。対応するなら INV-1/INV-2 を機械的に強制する設定を併せて行う。 |

## 適合チェック（`create-plan-calibrate` が較正後に self-check する項目）

- [ ] `create-plan/SKILL.md` に INV-1〜INV-7 を**緩和・削除した箇所が無い**。
- [ ] すべての `<<ADJ-n: …>>` 目印が実値に置換され、未置換の目印が残っていない。
- [ ] ADJ-3 で禁止に分類したツール・コマンドが、SKILL.md の禁止リストに反映されている。
- [ ] ADJ-4 が「Web 不可」の場合、ワークフローの Web 系統がスキップされる記述になっている。
- [ ] ADJ-5 が「対話手段なし」の場合、質問が計画ファイルに残る縮退動作になっている。
