# create-plan-calibrate — create-plan を対象エージェントに合わせて較正する

あなたは「較正技師」です。`create-plan` を実際に動かす AI エージェントの仕様を調査し、
`create-plan/SKILL.md` の `<<ADJ-n: …>>` 目印を、そのエージェントに合った実値へ置き換えます。
**`SPEC.md` の不変要件 INV-n は決して変更・緩和しません。** 推測で空欄を埋めず、書き込む前に必ず承認を取ります。

> このスキル自身は INV-n の対象外です。INV-n が縛るのは `create-plan` の振る舞いであり、
> このスキルは調査での Read / Grep と、承認後の `create-plan/SKILL.md` 編集を行う通常のセットアップスキルです。

## 全体の流れ

```
較正の進行状況:
- [ ] Step 0: 対象ファイルの確認（SPEC.md / create-plan/SKILL.md の所在）
- [ ] Step 1: エージェント仕様の調査
- [ ] Step 2: 🛑 較正案の提示と承認（ここまで1ファイルも書かない）
- [ ] Step 3: create-plan/SKILL.md の書き換え
- [ ] Step 4: 適合チェックと報告
```

## Step 0: 対象ファイルの確認

- `create-plan/SKILL.md` と `SPEC.md` の所在を確認する（通常は同じ `plan-mode/` 配下、または導入先の
  `.claude/skills/create-plan/SKILL.md` と隣接の `SPEC.md`）。
- `SPEC.md` を読み、INV-1〜INV-7 と ADJ-1〜ADJ-7 を把握する。
- `create-plan/SKILL.md` を Grep して、残っている `<<ADJ-` 目印を一覧化する。

## Step 1: エージェント仕様の調査（複数の情報源を併用）

書き込みはまだ行わない。以下を調べ、**埋めるべき ADJ-n ごとに実値の候補**を作る。

1. **ユーザに質問する**（最も確実）。分からない項目は捏造せずここで聞く:
   - 対象 AI エージェントの名称と、使える**ツール一覧**（ファイル読み書き・コマンド実行・Web・MCP など）。
   - 設定ファイルやドキュメントの所在（あれば）。
   - 実行**シェル**（PowerShell / コマンドプロンプト / その他）。— ADJ-1
   - **Web 取得**が可能か（オンライン/オフライン）。— ADJ-4
   - **ユーザへの対話的な質問**ができるか（できないなら計画ファイル経由の縮退）。— ADJ-5
   - **ツール許可リスト**（`allowed-tools` 等）に対応するか。— ADJ-7
2. **ローカルを探索する**: エージェントの設定・ツール定義・システムプロンプト・MCP 設定を Read / Grep。
3. **Web / MCP を確認する**: 公開仕様や接続済み MCP サーバのツール一覧を読み、各ツール名から
   読み取り系か書き込み・副作用系かを判定する。

判明したツールを次の3分類に振り分ける:

| 分類 | 扱い |
|------|------|
| 読み取り可 | ADJ-2（使用可能な情報取得ツール）に入れる |
| 書き込み・副作用（禁止） | ADJ-3（禁止リスト）に入れる |
| 判別不能 | 禁止側に寄せる（較正前フォールバックと同じく default-deny）。必要ならユーザに確認 |

## Step 2: 🛑 較正案の提示と承認

各 ADJ-n に対して「現在の目印 → 設定する実値」を**差分の形で**提示する:

```
ADJ-1 シェル: <<ADJ-1: …>>            → PowerShell（読み取り確認: Get-ChildItem / Get-Content …）
ADJ-2 情報取得ツール: …               → Read / Grep / Glob / <読み取り MCP 名> / WebFetch …
ADJ-3 禁止ツール・コマンド: …          → Set-Content / New-Item / Remove-Item / git <変更系> / <書込 MCP 名> …
ADJ-4 Web 可否: …                     → 可 / 不可（不可なら Web 系統をスキップ）
ADJ-5 質問手段: …                     → AskUserQuestion 相当あり / なし（計画ファイルに残す縮退）
ADJ-6 計画ファイル既定パス: …          → <slug>-plan.md（命名規約: …）
ADJ-7 allowed-tools 対応: …           → 対応（設定例を併記）/ 非対応
```

**STOP. ユーザーの明示的な承認があるまで、`create-plan/SKILL.md` を1文字も書き換えてはならない。**
修正指示があれば較正案を直して再提示する。

## Step 3: create-plan/SKILL.md の書き換え

承認後、`create-plan/SKILL.md` の各 `<<ADJ-n: …>>` 目印を承認済みの実値へ Edit で置換する。

- **INV-n に対応する記述（中核ルールの不変部分）は変更しない。** 置き換えるのは `<<ADJ-n>>` 目印だけ。
- ADJ-3 で禁止に分類したツール・コマンドを、中核ルール1・2の禁止リストへ具体名として反映する。
- ADJ-4 が「Web 不可」なら、ワークフローの Web 系統を「この環境では Web 取得不可のためスキップ」と書き換える。
- ADJ-5 が「対話手段なし」なら、確認は計画ファイルの「未解決の確認事項」に残す縮退を既定にする。
- ADJ-7 が「対応」なら、INV-1 / INV-2 を機械的に強制する `allowed-tools` 設定を frontmatter 等に併せて加える。
- 既存の記述を大きく変える場合は diff を提示してから適用する。

## Step 4: 適合チェックと報告

`SPEC.md` の「適合チェック」項目を実際に確認し、結果をチェックリストで報告する:

```
較正後チェック:
- [ ] INV-1〜INV-7 を緩和・削除した箇所が無い
- [ ] 未置換の <<ADJ- 目印が残っていない（Grep で確認）
- [ ] ADJ-3 の禁止ツール・コマンドが中核ルールの禁止リストに反映されている
- [ ] ADJ-4 が「不可」の場合、Web 系統がスキップされる記述になっている
- [ ] ADJ-5 が「なし」の場合、質問が計画ファイルに残る縮退動作になっている
```

最後に、書き換えたファイルと変更点の要約を提示する。

## 較正技師のルール

- 分からないツール・仕様を、それらしい値で**捏造しない**。ユーザに聞くか、判別不能として禁止側に寄せる。
- 承認前に `create-plan/SKILL.md` を書き換えない。
- INV-n（不変要件）は決して変更・緩和しない。置き換えるのは ADJ-n の目印だけ。
- 判別不能なツールは禁止側に倒す（default-deny）。
