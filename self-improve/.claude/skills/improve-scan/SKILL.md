# improve-scan — 改善の種を発見して backlog に貯める（/improve-scan）

普通の単発セッションで出た**訂正・繰り返し・行き詰まり**や、**実態とスキル定義のズレ**を拾い上げ、
**改善候補（backlog）**としてローカルに記録するスキルです。**GitHub も git も使いません**。
ここでは**ファイルを一切変更しません**（適用は `/improve-apply`）。

## 使い方

- `/improve-scan` — 既定（回収キューに積まれたセッションを対象）
- `/improve-scan --queue` — 回収キュー（フックが積んだ対象）を明示
- `/improve-scan --days N` — 直近 N 日のトランスクリプトを走査（フォールバック）

「改善の種を集めて」「最近の会話を振り返って改善候補を出して」のような自然文でも発動します。

## 入力ソース（jsonl の特定）

- **基本は回収キュー**: SessionEnd フック（`si-session-end.sh`）が `transcript_path` を
  `~/.claude/self-improve/<project>/queue.tsv` に積む。improve-scan は**このキューから transcript パスを
  読む**（プロジェクトキーの再導出は不要）。
- **kb のキューも一級ソースにする（knowledge-share 連携）**: `~/.claude/knowledge/queue/pending-sessions.tsv`
  が**あれば**、自前キューと**併せて**読む。両キューとも TSV で **2列目が `session_id`**・**4列目が
  `transcript_path`** の同型スキーマなので、**`session_id`（2列目）で union・dedup** し、重複セッションは
  1件として扱う。kb が無ければ自前キューだけで動く（単体フォールバック）。
- **`--days N` のときだけ** `~/.claude/projects/*/` を期間で走査するフォールバックを使う。
- **横断ナレッジ（あれば）**: `~/.claude/knowledge/` のエントリも入力として読む（後述「昇格候補」）。

> **抽出スクリプトは共有しない**: kb の `kb-extract-candidates.sh` は「エラー→解決ペア」抽出が目的で、
> self-improve の検出対象（訂正/逸脱/スキルギャップ＋ツール呼び出し vs SKILL.md 突合）とは別物。
> improve-scan は**自前の検出**を使う。kb と共有するのは「**どのセッションを見るか**＝キュー」だけ。

## 2系統で分類する

### skills-evolve（成功ルート＝客観突合）

既存スキルを使った作業で、**ツール呼び出し履歴とその SKILL.md のズレ**を検出する。AI の自問ではなく
**実行ログの差分**を根拠にする（嘘のない改善）。

1. transcript jsonl を **Skill 起動でセグメント分割**する
   （`type:user` の先頭が `/<skillname>` の行、または Skill ツール呼び出しを区切りに使う）。
2. 各セグメント内の `Bash`/`Edit`/`Read` 等ツール呼び出し列を、当該スキルの SKILL.md「フロー」見出しの
   手順と突合する。
3. **スキップされた手順**（条件付き実行への変更候補）／**文書に無い手順を実行**（新フェーズ追記候補）を拾う。

### skills-learn（失敗ルート＝訂正から学習）

- 訂正（「そうじゃなくて」「やり直して」「違う」）/ 繰り返し指示（毎回同じ依頼）/ 行き詰まり・摩擦 /
  エラーを自力回避した痕跡 / 外部レビューの指摘、を拾う。

## 補助ルート（棚卸しと効果検証）

### 棚卸し（鮮度管理＝「減らす」方向の改善）

スキャン対象のトランスクリプト全体で**一度も起動されていない自作スキル**（対象プロジェクトの
`.claude/skills/` 配下。プラグイン由来は対象外）を「棚卸し候補」として backlog に挙げる。

- 根拠はログの客観事実のみ: Skill 起動・`/<skillname>` 呼び出しが1件も無いこと。日数ではなく
  「今回のスキャン範囲で未使用」を明記する（例: 「直近14日・12セッションで起動 0 回」）。
- 提案は削除ではなく **(a) アーカイブ（`.claude/skills-archive/` へ移動）** か
  **(b) description のトリガー改善（発動しないのが問題の場合）** の2択で出す。採否はユーザーが判断。
- 先行例: UniM0cha/claude-self-improving-skills の stale 管理（30日 stale / 90日アーカイブ）の
  承認制版（self-improve の RESEARCH.md 参照）。

### 効果検証（適用済み改善の再発突合）

improve-apply が採用履歴に残す「**期待する効果**」（どのシグナルが消えるはずか）を読み、
**適用より後の**セッションで同種のシグナルが再発していないかをログで突合する。

- 再発していれば「効果なし: <改善内容> → 巻き戻し or 再改善」を backlog に挙げる
  （根拠: 適用日時と再発したセッション/シグナル）。再発が無ければ何も出さない（沈黙＝成功）。
- 「期待する効果」が記録されていない古い適用項目はスキップする。
- 先行例: Darwin Gödel Machine の実証検証・PACE の受理判定を「人間承認＋ログ突合」で置き換えた
  ローカル版（self-improve の RESEARCH.md 参照）。

## シグナルと構造化

- シグナル種別: 訂正 / 繰り返し / 摩擦 / **機能ギャップ（→新スキル候補）** / 成功技法（→定着候補）/
  ワークフロー選好（→CLAUDE.md・rules 候補）。
- 各候補に **頻度・一貫性（HIGH/MED/LOW）・定型ステップ割合** と **WHAT/HOW/FLOW** を付ける
  （FLOW 内の WHAT/HOW は FLOW 側だけで数え、二重カウントを避ける）。
- 各候補に **根拠（どのセッション/位置で何が起きたか）** を添える。

## 昇格候補（kb 連携・あれば）

`~/.claude/knowledge/` がある場合、シグナル抽出に加えて次を「昇格候補」として backlog に挙げる:

- **`#promote` 付きの kb エントリ**: `~/.claude/knowledge/index.md` で `#promote` タグの付いた行
  （kb-harvest やユーザーが「反復・ワークフロー級」と判断したもの）。
- **ログ上で再発しているのに kb に既存エントリがあるもの**: 同じ問題が直近トランスクリプトで再び
  起きていて、対応する kb エントリが見つかる場合。**再発回数は improve-scan がログから数える**
  （kb 側のファイルは書き換えない）。

これらは backlog に「**昇格候補: KB-… を `.claude/rules`/skill/CLAUDE.md へ昇格**（根拠: N 回再発 /
`#promote` 付与）」として記録する。実際の昇格と kb への書き戻しは `/improve-apply` が行う。
`#promoted`（昇格済み）や `- 昇格:` が既にあるエントリは候補にしない（再提案防止）。

## 出力先（配置を確定）

```
~/.claude/self-improve/<project>/improvement-backlog.md
```

- **`<project>` キーの算出はフック・スキル共通**（一致させる）:
  1. `cwd` を**プロジェクトルートに正規化** = `.claude/` か `CLAUDE.md` を持つ最も近い上位ディレクトリを
     上方探索（無ければ `cwd`。`git` には依存しない）。
  2. その正規化パス `root` に対し `printf '%s' "$root" | cksum | cut -d' ' -f1`（POSIX `cksum`）。
  - サブディレクトリで実行しても同一キーに収束する。
- **リポジトリ外に置く**のでチームへのコミット混入が無く、**プロジェクト単位**なので複数案件が混ざらない。
- **サニタイズ必須**: 生ログ・絶対パス・秘密情報（トークン/内部ホスト名/顧客データ）を backlog に書かない。
  根拠は「どのスキルのどの手順」「何回起きたか」など**一般化した形**で残す。
- **却下済み/合法的例外**（apply が記録）を読み、**同じ候補を再提案しない**。

## このスキルがやらないこと

- ファイルを編集する（発見のみ。適用は `/improve-apply`）
- 生ログ・秘密情報を backlog に書く（必ずサニタイズ）
- git / GitHub を使う（ローカル完結）
