# ナレッジベース（kb・Codex 版）

複数セッション・複数リポジトリで解決した知見を `~/.codex/knowledge/` に貯め、次に同じ問題で即座に引く。

- **インデックス** `~/.codex/knowledge/index.md` … 1エントリ＝1行。
  **Codex には `@import` 自動読込が無い**ため、横断索引の自動注入は限定的。主要エントリの要約を
  `AGENTS.md`（repo 直下）の索引断片に置くと、Codex がプロジェクト指示として取り込む（auto-load の限定的代替）。
- **トピック本体** `~/.codex/knowledge/topics/<topic>.md` … 本体（kebab-case・オンデマンド）。索引は 200行/25KB 以内。

> CC 版との差分は SPEC K2 の対応表を参照。中核の書式・3モード・サニタイズは同一。

## モード1: 記録（既定）

重複チェック（index）→ topic 選定 → 本体を下記書式で追記 → index に1行 → 予算管理（古い行は archive へ）→
AGENTS.md の索引断片を更新（auto-load の限定代替・任意）。

```
## KB-20260612-01: <一行タイトル>
- 日付 / 出典: <リポジトリ名>（session: <id 先頭8桁>）
- 環境: <ツール・バージョン>
- 問題: <他リポジトリでも起こり得る一般形で>
- 原因: / 対処: / 物証: <核心1行>
- タグ: #git #docker
```

## モード2: 検索（kb search <語>・読み取り専用）

`grep -i "<語>" ~/.codex/knowledge/index.md` → ヒットした topic を `grep -rin` で確認 → 対処を要約提示。無ければ「既存ナレッジには無し」。

## モード3: 昇格（kb promote）

プロジェクト固有の知見を一般形に書き換えて横断ナレッジへ。固有の規約・ディレクトリ名・ビジネスルールは昇格しない。

## ハードルール

- **機密サニタイズ**: トークン・内部ホスト名・IP・顧客データ・個人情報を記録しない。エラーは核心1行のみ。
- **一般化 / 小さく保つ / 重複させない**: 本文は再利用できる形に。固有情報は「出典」欄へ隔離。index は予算内。記録前に index 確認。

## 非対応（SPEC K3）

`kb-harvest`（過去ログ採掘）・`improve-scan` / `improve-apply`（自己改善）は **Codex のセッションログ＋フック機構に依存**するため
本増分では非対応。代替として、機密非コミット・サニタイズ等の規範を `AGENTS.md` に明記して補う。Codex のセッションログ仕様が
確認できたら harvest/scan を追加する。
