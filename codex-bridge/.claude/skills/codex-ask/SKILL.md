# codex-ask — Codex に相談する／セカンドオピニオン（/codex-ask）

設計の是非・代替案・デバッグ方針・トレードオフなど、**自由形式の相談**を
**OpenAI Codex** に投げて意見をもらうスキルです。ユーザーは Codex を直接操作しません。
Claude Code が Codex CLI を**非対話モード**（`--sandbox read-only`）で駆動し、Codex の
回答を要約して提示します。実際の codex 実行は **`codex-advisor` サブエージェント**に
委譲し、冗長な出力をメインの文脈から隔離します。

## 使い方

- `/codex-ask この設計（〜）は妥当？代替案は？`
- `/codex-ask なぜこのテストが落ちるか方針を相談したい @path/to/test`

「Codex に相談して」「codex の意見も聞いて」のような自然文でも発動します。

## 他スキルとの違い

- **codex-review** … 差分/コードの**指摘**（重大度 P1–P4）。「悪いところを挙げて」。
- **codex-ask** … 自由形式の**助言・議論**。「どう設計すべき？」「この方針は妥当？」。
- **codex-implement** … コードを**書かせる**。codex-ask は read-only で書き換えない。

## 前提

`codex` CLI が導入・認証済みであること（`codex-bridge/README.md` の「前提」参照）。
未導入・未認証の場合、サブエージェントが日本語で案内して終了します。

## フロー

1. **相談内容を整理する** — 何を判断したいか、選択肢、制約を明確にする。
2. **明らかに関係するファイル内容を用意する** — 相談に必要な抜粋を渡し切る
   （大きすぎる場合は関連箇所に絞る）。
3. **`codex-advisor` を起動する**（Task ツール） — 相談内容と同梱ファイルを渡す。
   サブエージェントが Codex を read-only で実行し、回答要約を返す。
4. **結果を提示する** — 結論・根拠・推奨アクション・留意点を見せる。
5. **必要なら橋渡しする** — 方針が固まったら `/codex-implement`（実装）や
   `/codex-review`（レビュー）に続けることを提案する。

## このスキルがやらないこと

- ユーザーに codex を手動で叩かせる（すべて Claude が非対話で駆動する）
- リポジトリのファイルを編集する（read-only。書かせたいなら `/codex-implement`）
